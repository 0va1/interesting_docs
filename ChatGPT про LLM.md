# Что нужно для создания собственной системы с мульти-агентами и RAG на слабом железе
На слабом железе необходима подписка на LLM API
## RAG
Для RAG не нужна LLM вообще. Для поиска используется CPU.
Но при создании Vector DB GPU существенно ускоряют работу. Например, при объёме сырых данных 1TB время создания на CPU может быть несколько суток. Но это делается один раз. К тому же если вначале объём данных не такой большой и постепенно увеличивается, то это будет даже незаметно
## Структура и модели

GPT-4o / Claude   ← ORCHESTRATOR 
Worker Agent     Planner Agent    RAG Agent 
(4o-mini)        (Claude)        (Gemini Flash) 
tools          vector DB        embeddings

🟢 Стартовый стек (оптимальный)
🧠 GPT-4o → главный оркестратор
🧠 Claude Sonnet → планирование сложных задач
⚙️ GPT-4o-mini → рабочие агенты
⚡ Gemini Flash → дешёвые RAG/lookup агенты

🟡 Более дешёвый вариант
GPT-4o-mini (всё)
иногда Claude для сложных задач

🔴 Premium (если делаешь продукт)
GPT-4o / GPT-5-class → orchestration
Claude Opus/Sonnet → reasoning layer
Gemini Pro → long-context RAG
mini models → workers

👉 в multi-agent системах не нужна одна мощная модель

Нужна иерархия моделей:

большой LLM → думает
маленькие LLM → делают работу
embeddings → ищут информацию

Короткий вывод

✔ GPT-4o → лучший “главный мозг”
✔ Claude → лучший “аналитик/планировщик”
✔ Gemini Flash → лучший “массовый дешёвый агент”
✔ mini-модели → для всего рутинного

Недостаток - LLM-API может быть урезана
Альтернатива - поднять свой сервер на облаке с GPU. Но это дорого, и только открытые модели.
